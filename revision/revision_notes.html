Revision Notes - Semantic Web & Knowledge Graphs

üîë Key Concepts

1. The Semantic Web

  • Web of Documents ‚Üí Web of Data
  • Goal: Make data machine-readable with embedded semantics
  • Uses W3C standards: RDF, RDFS, OWL, SPARQL

2. Knowledge Graphs

  • Graph-based knowledge representation
  • Nodes: Entities (resources)
  • Edges: Relationships (properties)
  • Examples: Google Knowledge Graph, Wikidata, DBpedia

üìä RDF (Resource Description Framework)

RDF Triples

Basic unit: (Subject, Predicate, Object)

  • Subject: Resource (IRI)
  • Predicate: Property/Relationship (IRI)
  • Object: Resource (IRI) or Literal (value)

Example:

ex:Bob foaf:knows ex:Alice .
ex:Bob schema:birthDate "1990-07-04"^^xsd:date .

Serialization Formats

  • RDF/XML: Original standard, verbose
  • Turtle: Human-readable, uses prefixes
  • N-Triples: Line-based, good for processing
  • JSON-LD: JSON extension for RDF
  • RDFa: Embedded in HTML

üèóÔ∏è RDFS (RDF Schema)

Purpose

Defines vocabularies - classes and properties for RDF data

Key Classes

  • rdfs:Class: Defines all classes
  • rdfs:Resource: Defines all resources
  • rdfs:Literal: Defines literal values
  • rdfs:Property: Defines all properties

Key Properties

  • rdfs:subClassOf: Class hierarchy
  • rdfs:subPropertyOf: Property hierarchy
  • rdfs:domain: Subject class constraint
  • rdfs:range: Object class/datatype constraint

Inference

  • Type Inheritance: If A subClassOf B, then instances of A are also B
  • Type Inference: Infer class from domain/range constraints

üéØ SKOS (Simple Knowledge Organization System)

Purpose

Create structured vocabularies, taxonomies, and thesauri

Core Elements

  • skos:Concept: Basic unit of thought
  • skos:prefLabel: Preferred label
  • skos:altLabel: Alternative label (synonym)

Relationships

  • skos:broader / skos:narrower: Hierarchical
  • skos:broaderTransitive: Explicit transitivity
  • skos:related: Associative (non-hierarchical)

Important!

skos:related is disjoint with skos:broader/narrower

ü¶â OWL (Web Ontology Language)

Purpose

Define complex ontologies with rich semantics and reasoning

Terminology

  • TBox: Terminology (ontology) - classes and properties
  • ABox: Assertions (data) - individuals/instances

Components

  • Classes: Concepts/types
  • Object Properties: Relationships between individuals
  • Data Properties: Attributes with literal values
  • Individuals: Instances of classes

Class Characteristics

  • owl:equivalentClass: Two classes with same instances
  • owl:disjointWith: No shared instances

Property Characteristics

  • Transitive: If A‚ÜíB and B‚ÜíC, then A‚ÜíC
  • Symmetric: If A‚ÜíB, then B‚ÜíA
  • Asymmetric: If A‚ÜíB, then NOT B‚ÜíA
  • Inverse: hasWife inverse of hasHusband

Reasoning Tasks

  1. Consistency Checking: No contradictions
  2. Concept Satisfiability: Can class have instances?
  3. Classification: Build inheritance hierarchy
  4. Realisation: Compute direct types of individuals

üîó Linked Data Principles

  1. Use IRIs to identify things
  2. Use HTTP IRIs so they can be resolved
  3. Provide useful information when IRI is looked up (RDF)
  4. Link to other data to enable discovery

üí≠ Key Distinctions

Open World vs Closed World

  • Open World (Semantic Web): Unknown = may be true or false
  • Closed World (Relational DB): Unknown = false

Schema vs Instance

  • Schema: Defines types and relationships (vocabulary)
  • Instance: Actual data conforming to schema

RDFS vs OWL

Feature RDFS OWL
Expressiveness Basic Rich
Constraints Limited Comprehensive
Reasoning Basic inference Advanced reasoning
Use Case Simple vocabularies Complex ontologies
